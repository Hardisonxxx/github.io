把Gridea部署在服务器上！！！

Denglzz 2022-03-21 22:22:00
Categories： 05 计算机 > 01 Linux Tags：

作为博客写作多年的老用户来说，这几年也是用遍各大博客框架，最后发现还是Gridea好用！ 但是Gridea作为本地的博客平台来说，它是在部署在github上的，众所周知Gayhub在国内的访问速度堪忧，正好本人有一个阿里云的服务器，所以俺就打算利用sftp将静态博客部署在服务器上。

1.使用ssh登录到服务器中，使用groupadd命令给sftp创建一个组（直接使用root恐怕有点怕怕哦）

groupadd sftp_users

2.添加用户到组sftp_users,并设置权限(创建新的用户gridea，然后添加至sftp_users中)

useradd -m -G sftp_users gridea

使用下面的chpasswd命令设置密码

echo "gridea:<输入密码>" | chpasswd

现在设置用户所需的权限：

chown root /home/gridea

在家目录中创建一个上传目录，并设置正确地所有权：

mkdir /home/gridea/upload
chown gridea /home/gridea/upload

3.编辑 sftp 配置文件 /etc/ssh/sshd_config

sftp 操作是通过 ssh 完成的，所以它的配置文件是 /etc/ssh/sshd_config，在做任何更改前，建议首先备份文件，然后再编辑该文件，接下来添加下面的内容：

cp /etc/ssh/sshd_config /etc/ssh/sshd_config-org
nano /etc/ssh/sshd_config


#注释这行Subsystem      sftp    /usr/lib/openssh/sftp-server

#末尾添加下面几行
Subsystem   sftp    internal-sftp
Match Group sftp_users
X11Forwarding no
AllowTcpForwarding no
ChrootDirectory %h
ForceCommand internal-sftp

重新启动 ssh 服务：

systemctl restart sshd

4.客户端配置

客户端配置完成以后点击保存，然后可以检测一下远程连接。没问题的话进入下一步。

5. Nginx的配置

由于本人的服务器已经搭建的宝塔面板，并且已经安装了nginx。所以直接在宝塔面板里进行设置。 在宝塔面板里打开nginx配置页面。 直接在nginx配置页面中默认的server{}上面复制下列代码。（然后安装下列<>中的提示更改自己的配置） （或者在终端中配置.conf文件都可以~）

server { listen 80; listen [::]:80; server_name <你的域名>; return 301 https://request_uri; } server { listen 443 ssl http2; listen [::]:443 ssl http2; server_name <你的域名>; charset utf-8;

# ssl配置
ssl_protocols TLSv1.1 TLSv1.2;
ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
ssl_ecdh_curve secp384r1;
ssl_prefer_server_ciphers on;
ssl_session_cache shared:SSL:10m;
ssl_session_timeout 10m;
ssl_session_tickets off;
ssl_certificate <pem文件的绝对路径>;
ssl_certificate_key <key文件的绝对路径>;  #pem和key文件属于ssl加密的文件，在可以在阿里云中获取免费一年的认证

root /home/gridea/upload;
location / {
}

}

然后保存即可

6.在Gridea本地客户端上点击同步，待同步成功后即可访问域名。
